Přístupnost

Přístupný web mohou používat i lidé s různými postiženími. Je to důležité téma, které se často opomíjí.

Co je webová přístupnost

Webová přístupnost (anglicky accessibility, zkráceně a11y) znamená, že web mohou používat všichni lidé bez ohledu na jejich schopnosti nebo omezení.

Přístupný web pomáhá lidem:

  • Se zrakovým postižením - nevidomí, slabozrací
  • Se sluchovým postižením - neslyšící
  • S motorickým postižením - nemohou používat myš
  • S kognitivním postižením - dyslexie, ADHD
  • S dočasným omezením - zlomená ruka, hluk

Zajímavosti

  • V ČR žije přes 1 milion lidí s hendikepem
  • Přístupný web je často lepší pro SEO
  • Státní weby musí být přístupné ze zákona
  • Přístupnost pomáhá i starším uživatelům

WCAG - Pravidla přístupnosti

WCAG (Web Content Accessibility Guidelines) je mezinárodní standard pro přístupnost. Aktuální verze je WCAG 2.2 a má čtyři hlavní principy.

1

Vnímatelnost

Informace musí být prezentovány tak, aby je uživatelé mohli vnímat.

  • Textové alternativy pro obrázky
  • Titulky u videí
  • Dostatečný barevný kontrast
2

Ovladatelnost

Rozhraní musí být ovladatelné různými způsoby.

  • Ovládání klávesnicí
  • Dostatek času na interakci
  • Žádné blikající prvky
3

Srozumitelnost

Obsah a ovládání musí být srozumitelné.

  • Čitelný text
  • Předvídatelné chování
  • Pomoc při vyplňování formulářů
4

Robustnost

Obsah musí být použitelný různými technologiemi.

  • Validní kód
  • Kompatibilita s asistivními technologiemi
  • Správné použití ARIA

Praktické tipy

Alt texty u obrázků

Každý významový obrázek potřebuje popis v atributu alt.

<img src="graf.png" 
     alt="Graf ukazující růst o 50%">

<!-- Dekorativní obrázek -->
<img src="dekorace.png" alt="">

Barevný kontrast

Poměr kontrastu mezi textem a pozadím musí být minimálně 4.5:1.

Špatně
Dobře

Ovládání klávesnicí

Vše musí jít ovládat bez myši. Focus stav musí být viditelný.

:focus {
    outline: 2px solid #264de4;
    outline-offset: 2px;
}

Struktura nadpisů

Nadpisy musí jít postupně H1 → H2 → H3, nesmí se přeskakovat.

<h1>Hlavní nadpis</h1>
  <h2>Sekce</h2>
    <h3>Podsekce</h3>

Formuláře

Každý input potřebuje propojený label.

<label for="email">E-mail:</label>
<input type="email" id="email">

Srozumitelné odkazy

Text odkazu musí dávat smysl i bez okolního kontextu.

✗ "Klikněte zde" ✓ "Stáhnout PDF ceník"

Jak testovat přístupnost

Ruční testování

  • Zkuste web ovládat jen klávesnicí (Tab, Enter)
  • Zvětšete stránku na 200%
  • Vypněte CSS a zkontrolujte strukturu
  • Vyzkoušejte screen reader (NVDA je zdarma)

Automatické nástroje

  • WAVE - rozšíření pro prohlížeč
  • Lighthouse - vestavěný v Chrome DevTools
  • axe - profesionální testování
  • Contrast Checker - kontrola kontrastů

Automatické nástroje zachytí jen asi 30% problémů. Ruční testování je vždy potřeba.

ARIA - Když HTML nestačí

ARIA (Accessible Rich Internet Applications) jsou speciální atributy, které pomáhají asistivním technologiím pochopit dynamický obsah.

Kdy používat ARIA

  • Interaktivní komponenty (tabs, modály)
  • Dynamicky měnící se obsah
  • Custom widgety (slidery, autocomplete)

První pravidlo ARIA: Nepoužívejte ARIA, pokud můžete použít nativní HTML element.

HTML ARIA příklady 
<!-- Popis pro screen reader -->
<button aria-label="Zavřít menu">
    ✕
</button>

<!-- Stav elementu -->
<button aria-expanded="false">
    Rozbalit
</button>

<!-- Live region -->
<div aria-live="polite">
    Položka přidána do košíku
</div>
